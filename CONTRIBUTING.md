# Contributing

Thank you for helping improve the **Aippy Creation Guidelines** documentation.

## Ways to contribute

- **Fix typos or unclear wording** in `docs/CREATION_GUIDELINES.md` or `README.md`.
- **Add translations** — prefer a `docs/translations/<locale>/` folder with a mirrored structure and a link from the main doc.
- **Update outdated sections** when the official product changes (cite sources: help center, announcements, or confirmed behavior).
- **Open an issue** for discussion before large reorganizations.

## Process

1. Fork the repository and create a branch from `main`.
2. Make focused changes; avoid unrelated edits.
3. Use clear commit messages (e.g. `docs: clarify Remix credit usage`).
4. Open a pull request describing **what** changed and **why**.

## Style

- Prefer **accessible, plain language** over jargon.
- Use **relative links** for files in this repo and **full URLs** for [aippy.ai](https://aippy.ai/) and example projects.
- Do not paste proprietary or confidential material from Aippy internals.

## Code of Conduct

All participants must follow the [Code of Conduct](CODE_OF_CONDUCT.md).
